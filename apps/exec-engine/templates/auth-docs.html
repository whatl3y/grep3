Overview
  • grep3 uses Ethereum signatures to protect existing repositories
  • Anyone can push to claim a new address/repo combination
  • Once claimed, subsequent pushes require proof of address ownership
  • Signatures are persistent and can be reused until revoked
How It Works
1 Get your challenge

Use the tool above or request a challenge via API. The challenge includes your address, repo name, and a nonce.

API
curl https://git.grep3.com/auth/challenge/0xYourAddress/repo-name
2 Sign the message

Sign the message field with your Ethereum wallet. The tool above handles this automatically.

3 Push with credentials

Use the generated credential as your git password (or username). The credential can be placed in either field.

0x{signature}
Terminal
git push grep3 main # When prompted: # Username: anything (or paste credential here) # Password: <paste credential>
Security
  • Signatures are persistent — sign once, push forever
  • Revoke signatures anytime by regenerating your nonce
  • Your private key never leaves your wallet
  • The server only verifies signatures, never stores them
First push

The first push to any new address/repo combination does not require authentication. This allows anyone to claim a repository. After the first push, all subsequent pushes require signature authentication.

Revoking signatures

If you believe your signature has been compromised, you can revoke all existing signatures using the revoke API. This regenerates your nonce, invalidating any previously signed credentials.

API Reference
GET /auth/challenge/:address/:repo
Returns a challenge to sign for authenticating pushes to an existing repository.

Parameters:
:address - Your Ethereum address
:repo - Repository name (with or without .git suffix)

Response (200): Challenge object with message to sign
Response (404): Repository not found (no auth needed for first push)
POST /auth/revoke/:address/:repo
Revokes all existing signatures for a repository by regenerating the nonce.

Parameters:
:address - Your Ethereum address
:repo - Repository name (with or without .git suffix)

Request body:
{ "signature": "0x..." } - Signed message "grep3:revoke:{address}:{repo}.git"

Response (200): Success + new challenge to sign
Response (401): Invalid signature
Response (404): Repository not found
